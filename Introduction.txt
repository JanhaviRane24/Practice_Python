What is Python?

Python is a high-level, interpreted, general-purpose programming language.
 High-level means I don't manage memory manually, interpreted means 
 code runs line-by-line without a separate compile step, and general-purpose 
 means it's used for web development, data science, automation, 
 scripting — not tied to one domain

 Variables, keywords, identifiers — know the distinction:

Variable — a name bound to a value (x = 5)

Keyword — reserved words Python itself uses (if, for, class, def) — 
can't be used as variable names

Identifier — any name you choose for a variable, function, or class — 
must start with a letter/underscore, no spaces, case-sensitive



2. Data Types & 3. Type Conversion

Core types: int, float, str, bool, complex

Type conversion functions: int(), float(), str(), bool() — convert between types.

Likely question: "What's the difference between implicit and explicit type conversion?"

"Implicit conversion happens automatically — like 1 + 2.5 becoming 3.5 because Python automatically promotes int to float. Explicit conversion is when I manually call int(), float(), str() to convert a value myself."

Likely question: "What does bool(0), bool(""), bool([]) return, and why?"

"All return False. Python treats 0, empty strings, and empty collections as 'falsy' by convention — any non-zero number, non-empty string, or non-empty collection is 'truthy'."

4. Operators

Arithmetic, Comparison, Assignment, Logical, Membership (in, not in), Identity (is, is not), Bitwise.

Likely question: "What's the difference between == and is?"

"== checks if two values are equal. is checks if two variables point to the exact same object in memory. Two different lists with identical contents are == but not is."

Likely question: "What's the difference between and/or and &/|?"

"and/or are logical operators, used for boolean logic and short-circuit evaluation. &/| are bitwise operators, operating on individual bits of integers — though they're also used for set operations and Series in pandas."

6. Loops

for i in range(5): print(i) — while i <= 5: ... i += 1 Control flow: break (exit loop entirely), continue (skip to next iteration), pass (do nothing, placeholder).

Likely question: "Difference between break and continue?"

"break exits the loop completely. continue skips the rest of the current iteration and moves to the next one, without exiting the loop."

Likely question: "What does range(1, 6) actually produce?"

"A sequence of numbers from 1 up to but not including 6 — so 1, 2, 3, 4, 5. range is memory-efficient because it generates numbers lazily rather than storing a full list."

7. Dictionary Methods

keys(), values(), items(), update(), get(), pop()

Likely question: "Difference between dict['key'] and dict.get('key')?"

"dict['key'] raises a KeyError if the key doesn't exist. dict.get('key') returns None instead (or a default value you specify), so it doesn't crash — safer when you're not sure the key exists."

Likely question: "What does a.update(b) do exactly?"

"It merges dictionary b into a — adding any new keys from b, and overwriting a's existing keys if b has the same key with a different value."

13. Exception Handling
python
try:
    # risky code
except SpecificError:
    # handle it
else:
    # runs only if no exception occurred
finally:
    # always runs, regardless of exception or not

Common exception types to know: ZeroDivisionError, ValueError, IndexError, TypeError, KeyError

Likely question: "What's the difference between except: (bare) and except SpecificError:?"

"A bare except: catches every possible exception, including ones you didn't anticipate, which can hide real bugs. Catching a specific exception type is better practice — you only handle what you expect, and let unexpected errors surface properly."

Likely question: "What's the difference between else and finally in a try block?"

"else only runs if the try block succeeded with no exception. finally always runs no matter what — exception or not — typically used for cleanup like closing a file or a database connection."

Custom exceptions:

python
class InsufficientBalanceError(Exception):
    pass

def withdraw(balance, amount):
    if amount > balance:
        raise InsufficientBalanceError("Not enough balance")

"You create custom exceptions by making a class that inherits from Exception, then use raise to trigger it — useful when you want a specific, descriptive error type for your own application logic, rather than relying on generic built-in exceptions."

Likely question: "Why is with open(...) preferred over manually calling open()/close()?"

"with automatically closes the file when the block ends, even if an exception occurs inside it — with manual open()/close(), forgetting to call close(), or an error happening before you reach it, leaves the file handle open, which can leak resources."

The Four Pillars — know these cold, they get asked constantly

1. Inheritance — one class acquires the properties and methods of another.

python
class Animal:
    def speak(self):
        print("Some sound")

class Dog(Animal):
    def speak(self):
        print("Bark")

super() — used to access the parent class's methods from inside a child class.

python
class Dog(Animal):
    def speak(self):
        super().speak()
        print("Bark")

2. Polymorphism — one interface, many forms. Achieved through method overriding.

"Different classes can define a method with the same name, but each implements it differently. When you call that method on an object, Python runs the version specific to that object's actual class."

3. Encapsulation — restricting access outside the class; wrapping data and methods into a single unit and controlling access to that data.

python
class Account:
    def __init__(self):
        self._protected = "can use by subclass/object"
        self.__private = "only functions in the class"
_single_underscore → protected (convention only — can still technically be accessed, but signals "don't touch this from outside")
__double_underscore → private (Python name-mangles it, making it much harder to access directly from outside)

4. Abstraction — hides implementation details, exposes only the necessary functionality.

"The user of a class doesn't need to know how a method works internally — just what it does. For example, you call .sort() on a list without knowing the sorting algorithm underneath."

Likely question: "Difference between Abstraction and Encapsulation, since they sound similar?"

"Encapsulation is about restricting access to data — using private/protected variables. Abstraction is about hiding complexity — showing only what's necessary and hiding the internal implementation, regardless of access level."

Likely question: "Difference between method overriding and method overloading?"

"Overriding: a subclass redefines a method that already exists in its parent class, with the same name and signature. Overloading (which Python doesn't support natively the way Java does) means multiple methods with the same name but different parameters — Python instead handles this with default arguments or *args."

17. Python's Memory Management

Reference counting: Python keeps track of how many variables reference a given object. When that count hits zero, the memory is freed.

Garbage Collector (GC): Handles a specific weakness of reference counting — circular references (two objects referencing each other, so their count never naturally reaches zero). The garbage collector periodically detects and cleans these up.

Namespaces: Built-in, Global, Local — determine variable scope and lookup order.

Likely question: "Why isn't reference counting alone enough?"

"If object A references object B, and B references A, their reference counts never drop to zero even if nothing else in the program uses them — that's a circular reference. Python's separate garbage collector specifically detects and cleans up these cycles that reference counting alone can't catch."

18. Iterators and Generators

iter(), next(), yield

Likely question: "What's the difference between an iterator and a generator?"

"A generator is a simpler way to create an iterator — using a function with yield instead of manually implementing __iter__ and __next__. Every generator is an iterator, but not every iterator is a generator."

Likely question: "What does yield actually do, differently from return?"

"return exits the function completely and gives back one value. yield pauses the function, remembers exactly where it left off, and gives back one value at a time — the next call to next() resumes right after the yield. This means generators produce values lazily, one at a time, instead of building the whole result in memory at once."

python
def count_up_to(n):
    i = 1
    while i <= n:
        yield i
        i += 1

for num in count_up_to(5):
    print(num)

20. Decorators (Basic)

"Functions are objects in Python — they can be passed around, assigned to variables, and passed as arguments to other functions. A decorator is a function that takes another function as input and extends or modifies its behavior without changing its actual code."

python
def my_decorator(func):
    def wrapper():
        print("Before the function runs")
        func()
        print("After the function runs")
    return wrapper

@my_decorator
def say_hello():
    print("Hello")

say_hello()

Likely question: "Where have you actually seen a decorator used?" This connects directly to your project — a strong, honest answer:

"In my DRF project — @api_view(['GET', 'POST']) and @permission_classes([IsAuthenticated]) are both decorators. They wrap my view function to add behavior — restricting HTTP methods, or checking authentication — before my actual function code runs."

21. Regular Expressions (re)

match() — checks for a match only at the beginning of the string. search() — checks for a match anywhere in the string, returns the first one. findall() — returns all matches in the string as a list.

python
import re
re.search(r"\d+", "My number is 12345")   # finds "12345"
re.findall(r"\d+", "a1 b22 c333")          # ['1', '22', '333']

Likely question: "Difference between match() and search()?"

"match() only checks the very start of the string — if the pattern isn't right at the beginning, it returns None, even if it exists further in. search() scans the entire string and returns the first match wherever it occurs."

22. Important Built-in Functions

len(), range(), sum(), max(), min(), sorted(), all()

Likely question: "Difference between sort() and sorted()?"

"list.sort() modifies the list in place and returns None. sorted(list) returns a new sorted list, leaving the original unchanged — and it works on any iterable, not just lists."

Likely question: "What does all() do?"

"Returns True only if every element in an iterable is truthy — useful for checking a condition holds across an entire collection, e.g. all(x > 0 for x in nums)."
